# 과제 수행 보고서

### 1. 과제를 어떻게 분석 하셨나요?

이번 과제에는 AI 챗봇을 구현해야 하는 구체적인 상황이 주어져 있었고, 특히 고객 데모 시연이 필요한 시급한 상황이라는 점에 주목했습니다. 제가 정말로 그 현장에 놓여 있다고 가정하며 몰입해 보았습니다. 분석 결과, AI 챗봇 서비스의 요구사항을 3시간 이내에 모두 완벽하게 구현하기에는 양이 적지 않다고 판단했습니다. 따라서 구현 순서의 우선순위를 정하는 것이 무엇보다 중요했습니다.

우선순위의 기준은 현재의 상황과 목표에 맞췄습니다. 3시간 내에 서비스를 구현해 고객에게 시연해야 하는데, 해당 고객은 API 스펙에 대한 깊은 이해보다는 AI를 통해 무엇을 할 수 있는지를 확인하는 것이 주된 목표였기 때문입니다. 이에 따라 첫 번째로 구현 범위를 조정했습니다. 3시간 이내에 완성도를 높일 수 있는 범위인 인증/인가, 대화 및 스레드 기능을 핵심 목표로 잡았고, 피드백이나 분석 보고 기능은 시간이 남을 경우 구현하는 선택 사항으로 분류했습니다. 실제 계획을 세울 때도 plan 1\~6까지는 핵심 기능과 문서화에 집중하고, plan 7\~8에 우선순위가 낮은 기능을 배치했습니다.

두 번째로는 고객 시연의 효과를 극대화하기 위해 요구사항에는 없었지만 간단한 데모 UI를 함께 구현하기로 했습니다. API 스펙만으로는 고객에게 대화 기능의 가치를 직관적으로 전달하기 어렵다고 판단했기 때문입니다. 비록 시간이 부족해 피드백 및 분석 보고 기능은 엔티티 설계 단계에 그쳤지만, 시연을 위한 준비만큼은 부족함이 없도록 최선을 다했습니다.

### 2. 과제의 진행함에 있어 AI를 어떻게 활용 하셨나요? 어떤 어려움이 있었나요?

과제를 진행하는 과정에서 AI는 단순한 도구 이상의 역할을 수행했습니다. 우선 과제 계획을 세우는 단계부터 AI와 협업했습니다. 프로젝트 개요와 요구사항을 바탕으로 구현 목표를 명확히 하고, 이를 점진적으로 개발할 수 있도록 1단계부터 8단계까지의 세부 계획(plan.md)을 정의하는 데 AI를 활용했습니다.

실제 코드 구현 단계에서도 AI의 도움을 받았습니다. 하나의 Plan 단위로 기능을 구현하도록 명령하고, 결과물이 나오면 제가 직접 검토한 뒤 커밋하는 과정을 반복했습니다. 특히 검토를 용이하게 하기 위해 각 단계가 끝날 때마다 docs 폴더에 작업 내용을 정리하도록 시켰는데, 이 덕분에 방대한 코드를 보다 빠르게 이해하고 관리할 수 있었습니다. 마지막으로 문서화 작업에도 AI를 적극 활용하여 작업 요약, API 성공 결과 기록, 그리고 README와 데모 가이드 작성까지 효율적으로 마칠 수 있었습니다.

다만 AI를 활용하면서 컨텍스트 관리와 그에 따른 토큰 소모 문제는 큰 어려움이었습니다. 모든 작업을 하나의 컨텍스트 내에서 관리하다 보면 학습량이 많아질수록 성능이 저하되는 노이즈 현상이 발생했습니다. 이를 방지하기 위해 컨텍스트를 초기화하고 다시 학습시키는 과정을 반복해야 했는데, 이 과정에서 많은 양의 토큰이 사용되어 제한된 자원을 관리하는 데 애를 먹기도 했습니다.

### 3. 지원자가 구현하기 가장 어려웠던 1개 이상의 기능을 설명 해주세요.

구현 과정에서 가장 기술적인 고민이 깊었던 부분은 실시간 스트리밍 답변의 순서 보장 로직이었습니다. SSE(Server-Sent Events)를 통해 응답을 보낼 때, 네트워크 환경이 불안정하면 토큰 전송 순서가 뒤바뀌어 사용자 화면에 문장이 엉켜서 출력되는 문제가 발생할 수 있었기 때문입니다.

이를 해결하기 위해 서버 측에서 각 토큰을 보낼 때마다 0부터 시작하는 고유 인덱스를 부여하도록 설계했습니다. 그리고 클라이언트에서는 단순히 데이터를 이어 붙이는 것이 아니라, 수신된 데이터를 Map 구조에 담아 인덱스 순서대로 매번 재정렬하여 출력하도록 로직을 개선했습니다. 이 과정을 통해 물리적인 도착 순서에 상관없이 AI가 생성한 논리적인 순서대로 데이터가 재조합되도록 유도했습니다.

하지만 실제 네트워크가 극도로 지연되거나 일부 패킷이 유실되는 극단적인 상황에서도 모든 데이터가 완벽하게 순서를 유지하며 렌더링될지에 대해서는 아직 확신이 없습니다. 이 부분은 향후 더 고도화된 예외 처리와 타임아웃 전략 등을 통해 계속해서 보완해 나가야 할 과제라고 생각합니다.
